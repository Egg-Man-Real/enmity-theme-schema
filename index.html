Enmity theme specification

Type: object

Credits for color descriptions goes to NEO#7154/compilekaiten.

Most pattern properties are there so the schema can be described fully; any key without a description should be considered "usage unknown".

 No Additional Properties

root $schema
Type: enum (of string)

Must be one of:

  • "https://beerpiss.github.io/enmity-theme-schema/enmity-theme.schema.json"

root name
Type: string

Name of the theme.

root author
Type: string

The author who made the theme.

root theme_color_map
Type: object

Properties that themes text and most background elements. All of these support different light mode and dark mode colors.

 No Additional Properties

root theme_color_map KEYBOARD
Type: array

Global keyboard background color

Must contain a minimum of 2 items

Must contain a maximum of 2 items

Each item of this array must be:

root theme_color_map KEYBOARD color
Type: object

  • hex_color
  • rgba_color
  • transparent_color
root theme_color_map KEYBOARD KEYBOARD items oneOf hex_color
Type: string

While Enmity does take #RRGGBBAA values, the alpha value is ignored when converting.

 Must match regular expression: #([0-9a-fA-F]{8}|[0-9a-fA-F]{6})
Example:

"#2E3440"
root theme_color_map KEYBOARD KEYBOARD items oneOf rgba_color
Type: string

Similar to CSS's rgba(), but Enmity only accepts rgba(r, g, b, a), not rgba(r g b a) or rgba(r g b / a).

 Must match regular expression: rgba\(((25[0-5]|2[0-4]\d|1\d{2}|0?\d?\d),\s*){3}(1|0|0?\.\d+)\)
Example:

"rgba(255, 255, 255, 0.04)"
root theme_color_map KEYBOARD KEYBOARD items oneOf transparent_color
Type: string

Special keyword to mark a color as fully transparent.

 Must match regular expression: ^transparent$
Example:

"transparent"

Examples:

[
    "#2E3440",
    "#ECEFF4"
]

[
    "rgba(46, 52, 64, 1)",
    "rgba(236, 239, 244, 1)"
]
